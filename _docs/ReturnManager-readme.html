PURPOSE

It's difficult to run a store efficiently without a mechanism for accepting, tracking, and processing returns.

Return Manager 1.0.1 Is a return or cancellation management system for Zen Cart to allow your customers or admin to ether cancel or return an individual item from orders.

Return Manager 1.0.1 was designed and tested with Zen Cart 1.5.5e only at this time. It should still work for 1.5.4 and up.

New Installation For Zen Cart 1.5.5e only!

Brand New Installation Of Return Manager:

  1. Download and Unzip the package contents to a temp directory
  2. Change and Rename the YOUR_ADMIN inside the zc155e folder to match the name of your shop's admin folder name.. Let me repeat this as this is the one step in a module install that most OFTEN gets overlooked/missed:

    Change and Rename the Admin_Folder inside the zc155e folder to match the name of your shop's admin folder name..
  3. Change and Rename all the YOUR_TEMPLATE folder names in the zc155e/includes folders to match the name of your template's folder name
  4. Once you renamed folders, rignt click the main zc155e folder and set the folder and file permissions to Read/Write for everyone

  5. Login to your store.
  6. Upload ALL the files in the zc155e folder to your store directory. The directory structure within these folders is like your Zen Cart store's folder structure if you have renamed them correctly. For most FTP clients the directory structure of the /includes directory should be preserved, and the files should "fall" into their proper places. If your FTP client isn't capable of this, you might have to manually place each file according to where they appear in this module.
  7. Click ANY link in the admin to trigger the auto-installer. This will add the necessary Tables, Fields and setup Return Manager in the Configuration Section.

HOW TO USE

Return Manager three basic ideas:

  • Customer Cancel ordered items - Store side login not required. Customer enters order number and email address for this order. If items have an order status below SHIPPED, then Cancellation is offered per-item!
  • Customer Return ordered items - Store side login not required. Customer enters order number and email address for this order. If items have an order status as SHIPPED or DELIVERED and within your return set days, then Return is offered per-item! An RMA number is issued for returns.
  • Admin Return/Cancel - Button in orders will allow admin to create an return or cancel for any item within this order.

Return Manager is based on simple rules, make it as easy as possible for customers or me to return/cancel items on a order! I didn't want to disturbed the order recored. Other programs work grate for that like Edit Orders. I wanted something that I could use along with existing mods so I'm not going to reproduce what's already done will.

Return or Cancellations rules! Your store, your rules.. If you do a search about returns you well find some interesting outlooks as to what works and what doesn't work. I tend to believe, the longer your return period, the better things work out. Some things just can't be returned like digital downloads. However, you can see if the item was downloaded or not and decide on refunds.. Order Status also plays into COWAA for digital downloads.

What better spot for product reviews or testimonial links then the order status page!!

Return Manager 24 on/off switches, please read!

Go to ADMIN->Configuration->Return Manager. There are 20 configurations that need to be looked at or set.. You need to set the two Status lines for returns and cancels.

  • Display Name field options set to: 0 = Display as Optional, 1 = Display as Required, 2 = Do not Display
  • Display Reason Text Area options set to: 0 = Display as Optional, 1 = Display as Required
  • Set Return Action Drop-down List. You can change this list to what your store requires! Your rules!
  • Set Cancel Action Drop-down List. You can change this list to what your store requires! Your rules!
  • Show Store Name, Address and Phone for Returns (ship to) on Returns Page. true=on false=off
  • Show Store Name, Address and Phone for Returns (ship to) on Returns Success Page. true=on false=off
  • Show Different Returns (ship to) Address on Returns Success Page. This is your address where the return well ship to.
  • Show comment notes! true=on false=off (This is the comments admin does in order status order page! Be careful on what you show them.)
  • Define Page options, affects return page.
  • Define Page options, affects cancel page.
  • Turn off the ability of Cancellations! Only Returns will work.
  • Admin Registered Customers Only? Only Registered Customers may submit a return request. true=on false=off
  • Admin Update Order Status option. true=on false=off Update Order Status upon RMA Successful submit.
  • Admin Order Status, The order status assigned when an RMA is submitted. You need to chance this to RMA# Issued
  • Admin Order Status for Delivered Items, The order status assigned when an order is (Complete - Shipped).
  • Admin Order Status for Canceled Items, The order status assigned when an order (Item is Canceled). You need to chance this to Cancel Item
  • Admin Return Grace Period Numeric Number Only! This represents your 30 Day Return Policy or how ever many days you allow a customer to return an item.
  • Admin Update Order Comments option. Update Admin Order Comments upon RMA Success. true=on false=off
  • Email RMA Grace Period, This tells your customer how many days till the RMA# expires.
  • RMA Divider, This is a divider for the RMA number set (ie.. 1015-5-FSE7D)
  • Admin Email Copy to admin! Send an copy of the customer to admin when mailed from the admin button.
  • RMA Divider. This is the divider used in the RMA Key creation.
  • RMA Key, This is a number of characters to use in the random generator (ie.. 5) is the default number used.
  • Return Manager Version, as with many of my mods, changing the version number reinstalls the SQL.

That's it you now have Return Manager Installed..

One thing to note, Install Edit Orders to farther enhance your store management.

Buttons! What do they do..
UPDATE Button below the address field, saves the return address to each item entry, red means that the fields are empty and you are looking at the default information.
UPDATE Button below the item list, updates the total items in this return or changes the type of return/cancel. Adding a 0 here deletes that item from the return and updates the total fields.
SAVE Button below the return total fields. Saves all the edit fields and total return dollars. Than updates the order status history with the total so its visible on the order or edit order page.
SEND MAIL Button sends this Return/Cancel information to your customer. You should edit the instructions to reflect your store requirements!!
